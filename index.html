Initial design.
  • Bars for each task extend until due date
  • Line marches along at current time
  • A notion of progress and time required can be visualized
  • Due-dates are kept as events in a special calendar
  • Extra information is stored in the notes field of that event
  • All information should be human-readable on the Google calendar interface
  • Handling oauth for the API may require storage of keys - can we do it in the browser's local store?
  • If at all possible, have no server component - do it all in the browser
  • Eventually have a way to edit and add tasks in the UI
  • Tasks may want to have subparts or start dates
  • Order and fade away tasks with the most important at the top
